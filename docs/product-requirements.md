# プロダクト要求定義書 (Product Requirements Document)

## プロダクト概要

### 名称
**Cat Live Wallpaper** - 手描き風の部屋でロシアンブルーが気ままに過ごすAndroidライブ壁紙

### プロダクトコンセプト
- **生活感のあるホーム画面**: 温かい手描き風の部屋を背景に、猫がさりげなく動くことで、ホーム画面に静かな生活感を生む。
- **最小構成で成立するライブ壁紙**: WallpaperService と Canvas 描画を軸に、まずは1匹の猫と1部屋で成立するMVPを目指す。
- **見た目と電池持ちの両立**: 見ていて心地よいアニメーションを保ちつつ、無駄な再描画を抑えてバッテリー消費に配慮する。

### プロダクトビジョン
Android のホーム画面を、単なる背景画像ではなく、眺めていて落ち着く小さな情景へ変える。
温かい手描き風の部屋の中で、グレーのロシアンブルーが歩いたり、座ったり、毛糸で遊んだりすることで、ユーザーに自然で穏やかな体験を提供する。
最初のリリースでは最小構成に絞り、安定動作と低負荷を優先しながら、将来的な昼夜差分や追加リアクションへ拡張しやすい土台を作る。

### 目的
- ホーム画面に温かく落ち着いた雰囲気を与えるライブ壁紙を提供する。
- 猫の自然な振る舞いにより、静的な壁紙にはない愛着と継続利用の動機を生む。
- Kotlin と WallpaperService を用いた最小構成で、実装と保守のしやすいライブ壁紙基盤を作る。

## 前提条件

### 技術的前提
- 対象プラットフォームは Android とする。
- 実装言語は Kotlin とする。
- ライブ壁紙の実装には WallpaperService を使う。
- 描画方式は Canvas ベースを前提とし、OpenGL と Compose は初期スコープに含めない。

### 対象デバイスと対応範囲
- 対象デバイスはスマートフォンを優先し、タブレット最適化は初期スコープ外とする。
- 画面向きは縦向き・横向きの両方に対応する。
- ホーム画面の左右スクロールに追従する壁紙オフセット対応を必須とする。

### MVPの固定制約
- 設定画面は後回しにし、まずは壁紙表示そのものを成立させる。
- 猫は最低限「歩く」「止まる」「遊ぶ」の3状態を持つ。
- 猫へのタップ反応はMVPに含める。
- 背景はMVPでは昼テーマ(`SceneTheme.DAY`)のみとし、P1での昼夜差分追加を見越した設計とする。
- バッテリー消費への配慮をMVPの必須要件とする。

## ターゲットユーザー

### プライマリーペルソナ: 佐々木美咲 (29歳、会社員)
- Android スマートフォンを日常的に使い、ホーム画面の見た目にこだわりがある。
- 猫が好きで、派手すぎない落ち着いたデザインやイラストテイストを好む。
- アプリを細かく設定するより、最初から雰囲気よく動く壁紙をそのまま使いたい。
- 一方で、ライブ壁紙は電池を消費しやすいという不安があり、重いものは避けたい。
- 朝や移動中、スマートフォンを開いたときに短時間でも癒やしを感じられるホーム画面を求めている。

## 成功指標(KPI)

### プライマリーKPI
- ライブ壁紙適用完了率: 70%以上(初回利用セッション内)
  - 壁紙プレビュー画面を開いたユーザーのうち、実際に適用まで完了した割合で測定する。
- 7日後継続利用率: 35%以上(初回公開後3ヶ月以内)
  - インストールから7日後もライブ壁紙が有効なユーザー割合を測定する。
- タップ反応体験率: 60%以上(初回利用セッション内)
  - 初回利用日に少なくとも1回タップ反応を発生させたユーザー割合を測定する。

### セカンダリーKPI
- クラッシュフリーセッション率: 99.5%以上(初回公開後3ヶ月以内)
- 壁紙プレビュー起動時間: 2秒以内(検証基準端末: Google Pixel 6a / Android 14 / 6GB RAM / 1080 x 2400)
- ユーザー満足度: ストア評価 4.2 以上(公開後3ヶ月以内)

### KPIの測定方法
- ライブ壁紙適用完了率: 壁紙プレビュー画面を開いてから適用完了までを1セッションとして扱い、MVP検証中は内部テストと手動集計で確認し、P1以降の匿名 analytics 導入後に継続測定へ移行する。
- 7日後継続利用率: インストールから7日後もライブ壁紙が有効である状態を継続利用と定義し、P1以降の匿名 analytics 導入後に正式測定する。
- タップ反応体験率: 初回利用日に少なくとも1回タップ反応が可視化された割合とし、MVP検証中は手動検証、P1以降の匿名 analytics 導入後に継続測定する。
- クラッシュフリーセッション率: Google Play Console の Android Vitals を主測定手段とする。
- 壁紙プレビュー起動時間: Google Pixel 6a のリリースビルドでローカル計測ログを用いて確認する。

## 機能要件

### コア機能(MVP)

#### ライブ壁紙の基本表示

**ユーザーストーリー**:
Androidユーザーとして、ホーム画面に温かい雰囲気を持たせるために、手描き風の部屋背景と猫が表示されるライブ壁紙を使いたい。

**受け入れ条件**:
- [ ] `background_room_day.png` を背景として表示できる。
- [ ] 背景上に猫を1匹表示できる。
- [ ] Android のライブ壁紙選択画面からプレビューと適用ができる。
- [ ] 縦向き・横向きの両方で表示が大きく崩れない。

**優先度**: P0(必須)

#### ホーム画面スクロール対応

**ユーザーストーリー**:
ホーム画面を左右に切り替えるユーザーとして、スクロールに合わせて壁紙の見える範囲も自然に動いてほしい。

**受け入れ条件**:
- [ ] `Engine.onOffsetsChanged()` を利用して背景の水平オフセットを更新できる。
- [ ] ホーム画面の左右スクロール量に応じて背景の切り出し位置が変化する。
- [ ] オフセット更新時に猫や背景の表示が破綻しない。
- [ ] オフセット更新が常時高頻度再計算の原因にならない。

**優先度**: P0(必須)

#### 猫の基本アニメーション

**ユーザーストーリー**:
猫のいる情景を楽しみたいユーザーとして、猫が静止画ではなく自然に歩いたり止まったり遊んだりしてほしい。

**受け入れ条件**:
- [ ] 猫が `WALK`、`IDLE`、`PLAY` の3状態を持つ。
- [ ] `WALK` では `cat_walk_1.png` と `cat_walk_2.png` を切り替えながら移動する。
- [ ] `IDLE` では `cat_sit.png` を表示して一定時間停止する。
- [ ] `PLAY` では `cat_play_1.png` と `cat_play_2.png` を切り替え、`toy_yarn.png` を表示できる。

**優先度**: P0(必須)

#### 猫の行動制御

**ユーザーストーリー**:
ホーム画面を見たときに単調さを感じたくないユーザーとして、猫の動きが一定の間を持って自然に切り替わってほしい。

**受け入れ条件**:
- [ ] 猫の状態が短時間で過剰に切り替わらないよう、各状態に最低滞在時間を設定できる。
- [ ] 歩行時は左右いずれかへゆっくり移動する。
- [ ] 猫の移動範囲は背景の後方エリア(後景移動範囲)に制限される。
- [ ] 状態遷移ロジックが描画処理から分離されている。

**優先度**: P0(必須)

#### タップ反応

**ユーザーストーリー**:
ライブ壁紙と少しだけ触れ合いたいユーザーとして、画面をタップしたときに猫が何らかの反応を返してほしい。

**受け入れ条件**:
- [ ] `Engine.onTouchEvent()` でタップ位置を取得できる。
- [ ] タップ位置に毛糸玉を一定時間表示できる。
- [ ] タップ後は通常遷移より `PLAY` を優先し、反応が視覚的に分かる。
- [ ] 追加素材なしで成立する最小反応として実装されている。

**優先度**: P0(必須)

#### 低負荷な更新制御

**ユーザーストーリー**:
ライブ壁紙を日常的に使うユーザーとして、見た目は楽しいまま、バッテリー消費や発熱はできるだけ抑えてほしい。

**受け入れ条件**:
- [ ] ホーム画面表示中のみ描画更新を行う。
- [ ] `IDLE` 時の描画更新頻度を `WALK` / `PLAY` 時より低く設定できる。
- [ ] Bitmap を毎フレーム再読込せず、初期化時に読み込んで再利用する。
- [ ] 画面サイズ変更時以外はスケーリングとレイアウト計算を再利用する。

**優先度**: P0(必須)

### 将来的な機能(P1/P2)

#### 夜背景の追加

昼だけでなく時間帯に応じた雰囲気の変化を求めるユーザー向けに、夜背景を追加する。MVPでは受け皿の設計まで行い、実際の切り替え演出は初期拡張として扱う。

**優先度**: P1(重要)

#### 設定画面

猫の速度、行動頻度、背景位置やスケールなどを調整できる設定画面を追加する。

**優先度**: P1(重要)

#### インタラクション強化

長押し、連続タップ、タップ地点までの自然な移動など、猫との触れ合いを増やす。

**優先度**: P1(重要)

#### 背景バリエーション拡張

天候差分、季節差分、複数の部屋テーマなどを追加し、長期利用時の飽きを減らす。

**優先度**: P2(できれば)

## 非機能要件

### パフォーマンス
- 検証基準端末: Google Pixel 6a (Android 14, 6GB RAM, 1080 x 2400)
- 壁紙プレビューの初回表示: 2秒以内(上記端末でのコールドスタート計測)
- `WALK` / `PLAY` 時の描画更新頻度: 最大 15fps
- `IDLE` 時の描画更新頻度: 最大 2fps
- 画面回転やサーフェスサイズ変更時の再レイアウト完了: 300ms 以内

### ユーザビリティ
- 初回利用者が説明なしで壁紙のプレビュー開始まで到達できる
- タップ反応は1回のタップで視覚的に理解できる
- デフォルト設定のままでも見た目が成立し、設定画面なしでも利用開始できる

### 信頼性
- 壁紙適用中にクラッシュしないことを優先し、クラッシュフリーセッション率 99.5%以上を維持する
- `onVisibilityChanged()` と `onSurfaceChanged()` に応じて更新開始・停止を正しく切り替えられる
- 画面の縦横切り替え後も背景と猫の表示が破綻しない

### セキュリティ
- ネットワーク通信や外部送信を前提としないローカル完結型とする
- タップ座標や利用状況を収集する場合は、識別可能な個人情報と結び付けない

### スケーラビリティ
- 背景テーマ差し替えを追加しても描画パイプラインの大規模な書き換えを必要としない
- 猫の追加状態や追加背景を導入しても、既存の描画責務と行動制御責務を維持できる

## スコープ外

明示的にスコープ外とする項目:
- タブレット向けレイアウト最適化
- Compose を使ったUI構成
- OpenGL / GLSurfaceView ベースの描画実装
- 複数匹の猫を同時に表示する機能
- ネット接続、クラウド同期、外部アカウント連携
